# Proteus v0.2 SP6 — TUI Panels — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the jobs panel and the designs panel to the TUI, and expand the chat-only layout to the multi-panel layout — so the user sees jobs and persisted designs (with their ipSAE) live.

**Architecture:** Two new Bubble Tea sub-models — `jobsModel` and `designsModel` — render the right-hand column. They are pure render-from-state: the root `Model` reloads `store.ListJobs` / `store.ListDesigns` on a one-second `tea.Tick` and pushes the slices into the panels. `View` composes chat-left + jobs/designs-right when the terminal is ≥100 columns wide, and falls back to a single `Tab`-cycled pane below that.

**Tech Stack:** Go 1.26, Bubble Tea v1.3, Lip Gloss v1.1, builds on SP1 (`store`/`domain`). Module path: `github.com/alvarogonjim/proteus`.

**Source of truth:** design doc `docs/superpowers/specs/2026-05-16-proteus-v0.2-design.md` (SP6); SPECS §10.2, §10.5.

**Conventions for every commit:** Conventional Commit subject; end commit messages with a blank line then `Co-Authored-By: Claude Opus 4.7 <noreply@anthropic.com>`. Run `gofmt -w` before committing. After each task, `go build ./...` and `go test ./...` must pass.

**Context:** SP1–SP5 are complete and merged. The TUI (`internal/tui`) has `theme.go`, `statusbar.go`, `chat.go`, `commandbar.go`, `modal.go`, `app.go` (the root `Model` — chat-only layout). `Model` already holds `store *store.Store`. `internal/store` has `ListJobs(domain.ProjectID)` and `ListDesigns(domain.ProjectID)`; `store.DefaultProjectID`. `internal/domain` has `Job` and `Design`.

**Refresh mechanism:** the design doc mentions event-driven panel updates; this plan uses a **1-second poll** of the store instead — simpler, no `jobs.Manager` callback wiring, and imperceptibly different for a human watching a TUI. The `jobs.Manager` `onUpdate` callback stays `nil`.

package tui

import (
	"fmt"
	"strings"

	"github.com/alvarogonjim/proteus/internal/domain"
)

// jobsModel renders the JOBS panel (SPECS §10.2).
type jobsModel struct {
	theme Theme
	jobs  []domain.Job
	width int
}

func newJobsModel(th Theme) jobsModel { return jobsModel{theme: th, width: 36} }

// setJobs replaces the panel's jobs.
func (m *jobsModel) setJobs(jobs []domain.Job) { m.jobs = jobs }

// setWidth sets the panel's render width.
func (m *jobsModel) setWidth(w int) { m.width = w }

// jobGlyph returns a single-rune status indicator for a job.
func jobGlyph(s domain.JobStatus) string {
	switch s {
	case domain.JobRunning:
		return "⟳"
	case domain.JobSucceeded:
		return "✓"
	case domain.JobFailed:
		return "✗"
	case domain.JobCancelled:
		return "⊘"
	default: // queued
		return "·"
	}
}

// shortID truncates an ID for compact panel display.
func shortID(id string) string {
	if len(id) > 10 {
		return id[:10]
	}
	return id
}

// View renders the jobs panel.
func (m jobsModel) View() string {
	var b strings.Builder
	b.WriteString(m.theme.StatusBar.Render("JOBS"))
	b.WriteString("\n")
	if len(m.jobs) == 0 {
		b.WriteString(m.theme.ToolTrace.Render("  no jobs yet"))
		return b.String()
	}
	for _, j := range m.jobs {
		line := fmt.Sprintf("%s %-18s %s %3.0f%%",
			jobGlyph(j.Status), j.Tool, shortID(string(j.ID)), j.Progress*100)
		b.WriteString(m.theme.ToolTrace.Render(line))
		b.WriteString("\n")
	}
	return strings.TrimRight(b.String(), "\n")
}
```

package tui

import (
	"fmt"
	"strings"

	"github.com/alvarogonjim/proteus/internal/domain"
)

// designsModel renders the DESIGNS panel — a table with the ID / pLDDT / ipSAE
// / ipTM / Lab columns (SPECS §10.2).
type designsModel struct {
	theme   Theme
	designs []domain.Design
	width   int
}

func newDesignsModel(th Theme) designsModel { return designsModel{theme: th, width: 36} }

// setDesigns replaces the panel's designs.
func (m *designsModel) setDesigns(designs []domain.Design) { m.designs = designs }

// setWidth sets the panel's render width.
func (m *designsModel) setWidth(w int) { m.width = w }

// score formats a design's named score, or "—" when it is absent.
func score(d domain.Design, key string) string {
	if v, ok := d.Scores[key]; ok {
		return fmt.Sprintf("%.2f", v)
	}
	return "—"
}

// View renders the designs panel.
func (m designsModel) View() string {
	var b strings.Builder
	b.WriteString(m.theme.StatusBar.Render(fmt.Sprintf("DESIGNS (%d)", len(m.designs))))
	b.WriteString("\n")
	b.WriteString(m.theme.ToolTrace.Render(
		fmt.Sprintf("  %-11s %6s %6s %6s %3s", "ID", "pLDDT", "ipSAE", "ipTM", "Lab")))
	b.WriteString("\n")
	if len(m.designs) == 0 {
		b.WriteString(m.theme.ToolTrace.Render("  no designs yet"))
		return b.String()
	}
	for _, d := range m.designs {
		line := fmt.Sprintf("  %-11s %6s %6s %6s %3s",
			shortID(string(d.ID)),
			score(d, "plddt_mean"), score(d, "ipsae"), score(d, "iptm"), "—")
		b.WriteString(m.theme.AgentText.Render(line))
		b.WriteString("\n")
	}
	return strings.TrimRight(b.String(), "\n")
}
```

func (m *Model) scheduleRefresh() tea.Cmd {

func (m *Model) reloadPanels() {

	if designs, err := m.store.ListDesigns(store.DefaultProjectID); err == nil {

}

## Self-review notes

- **Spec coverage:** the jobs panel (SPECS §10.2, criterion 8) — Task 1. The
  designs panel with the ipSAE column (criterion 7) — Task 2. The multi-panel
  layout, refresh, and `Tab` focus — Task 3.
- **Type consistency:** `jobsModel`/`designsModel` are value types with
  pointer-receiver `setJobs`/`setDesigns`/`setWidth`; `panelFocus` cycles
  `0..2`; `refreshMsg` is the tick payload; `shortID` is shared by both panels.
- **Refresh:** a 1-second `tea.Tick` reloads `store.ListJobs`/`ListDesigns` into
  the panels; the chain re-arms itself; a nil store (tests) is a no-op.
- **Known SP6 limitations (documented, not defects):** the WET-LAB panel area
  from the §10.2 mock is deferred to v0.4 (Adaptyv); panels are read-only (no
  in-panel selection/scrolling beyond the chat viewport); `Tab` focus only
  changes the visible pane in the narrow (<100 col) layout — in the wide layout
  all panels are always shown; refresh is poll-based, not event-driven.
