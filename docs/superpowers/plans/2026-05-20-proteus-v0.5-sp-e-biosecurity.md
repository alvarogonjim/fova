# Proteus v0.5 — SP-E "Biosecurity — Restricted-Target Guard" Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Refuse `design.*` and `lab.submit_experiment` tool calls whose **target sequence** matches a curated list of restricted, dual-use-concern biological sequences (a small subset of the Australia Group / US Select Agents toxin families). On a match the agent loop short-circuits before tool execution, posts a loud refusal in chat, and feeds the model the same refusal string so it stops re-trying. The list ships embedded in the binary; nothing is loaded from the network. SPECS §20 #3.

**Architecture.** A new `internal/safety/` package owns three concerns:

1. **The table.** `restricted_targets.toml` lives next to `restricted_targets.go` and is embedded with `//go:embed`. The Go side parses it once at startup into a `*Table` of `(id, reason, signatures[])` entries. The matcher normalises both sides (uppercase, strip whitespace) and asks: does the input sequence contain any signature as a substring? — case-insensitive `strings.Contains` per signature. (k-mer hashing is a v1+ improvement; the v0.5 scaling envelope is ~50 signatures × ~25 residues × inputs up to ~2000 residues, well inside the O(N·M) budget.)
2. **The guard.** `guard.go` defines `safety.Guard`, a small interface that the agent loop consults before executing any tool. It knows which tool names to inspect (`design.*`, `lab.submit_experiment`) and how to dig the target sequence out of each tool's input JSON. It is the single integration point: anything outside `internal/safety` only ever sees the `Guard` interface and a `Refusal` value.
3. **Wiring.** The guard is plumbed through `agent.NewLoop` (one extra argument; passed by `cmd/proteus/main.go`). `executeTool` calls `guard.Inspect(name, input)` between the confirmation check and `registry.Execute`; a refusal becomes a `ToolDoneMsg` with a non-nil `Err` and a `Display` that begins `refused by biosecurity guard:`, and the same string is fed back to the model. Option 1 from the spec — agent-loop choke-point — is chosen here because the loop already imports `internal/tools` and is the single place every tool call passes through; adding a field to `tools.Registry` would force every registry consumer (including tests in unrelated packages) to think about the guard.

**Refusal UX.** Loud and plain. No euphemism. The user sees a styled error line in the chat that contains the matched entry's `reason` verbatim — e.g. `refused by biosecurity guard: Select agent; botulinum toxin family banned`. The agent sees the identical string as the tool result, so it has no incentive to retry with a tweak.

**Sample list.** The embedded TOML ships with two illustrative entries (a botulinum-toxin-A signature and a ricin signature) using publicly-known N-terminal residue runs from UniProt. A comment in the TOML file states explicitly that this list is **illustrative, not exhaustive**, and that production deployments must layer external screening (e.g. IGSC's screening framework, vendor screening, NTI's common-mechanism work).

**Tech Stack:** Go, `github.com/BurntSushi/toml` (already a direct dep), `embed`, standard library only otherwise.

**Spec:** `docs/superpowers/specs/2026-05-20-proteus-v0.5-design.md` §6 (SP-E), §1, §9, §10.

**Branch:** `feat/v0.5-sp-e-biosecurity` — the implementer creates it off `master`; do **not** branch off `feat/v0.5-design`. Verify with `git fetch origin && git checkout -b feat/v0.5-sp-e-biosecurity origin/master && git merge --ff-only origin/master`.

```go
// Package safety implements the v0.5 biosecurity guard: a curated, embedded
// list of restricted target sequences and a substring screen that the agent
// loop consults before executing any design.* or lab.submit_experiment tool
// call.
//
// The list ships embedded in the binary (//go:embed) so the guard cannot be
// silently disabled by tampering with a config file or losing a network
// fetch. The match is a deliberately conservative case-insensitive substring
// screen; a hit returns a Refusal with the entry's id and human-readable
// reason. The reason is surfaced verbatim to the user — loud and plain,
// no euphemism (SPECS §20 #3).
package safety

import (
	_ "embed"
	"fmt"
	"strings"

	"github.com/BurntSushi/toml"
)

// defaultTOML is the embedded curated list. See restricted_targets.toml for
// schema and provenance notes.
//
//go:embed restricted_targets.toml
var defaultTOML []byte

// Entry is one restricted-target family in the list.
type Entry struct {
	ID         string   `toml:"id"`
	Reason     string   `toml:"reason"`
	Signatures []string `toml:"signatures"`
}

// Refusal is what Check returns on a hit.
type Refusal struct {
	ID     string // the matched Entry.ID
	Reason string // the matched Entry.Reason (shown verbatim to the user)
}

// Table is a loaded restricted-target list with normalised signatures.
type Table struct {
	entries []Entry
	// normalised holds entries' signatures pre-uppercased and whitespace-
	// stripped, parallel to entries. Built once at construction time.
	normalised [][]string
}

// LoadDefaultTable parses the embedded restricted_targets.toml. It is called
// once at startup by cmd/proteus/main.go.
func LoadDefaultTable() (*Table, error) {
	return parseTable(defaultTOML)
}

// parseTable parses a TOML byte slice into a *Table. Exposed only for tests.
func parseTable(data []byte) (*Table, error) {
	var doc struct {
		Entry []Entry `toml:"entry"`
	}
	if err := toml.Unmarshal(data, &doc); err != nil {
		return nil, fmt.Errorf("safety: parse restricted_targets.toml: %w", err)
	}
	t := &Table{entries: doc.Entry, normalised: make([][]string, len(doc.Entry))}
	for i, e := range doc.Entry {
		sigs := make([]string, 0, len(e.Signatures))
		for _, s := range e.Signatures {
			n := normalise(s)
			if n == "" {
				continue // skip blank signatures defensively
			}
			sigs = append(sigs, n)
		}
		t.normalised[i] = sigs
	}
	return t, nil
}

// Entries returns the parsed entries (read-only — callers must not mutate).
func (t *Table) Entries() []Entry { return t.entries }

// Check reports whether seq contains any signature from any entry. The match
// is case-insensitive after stripping whitespace from seq. On a hit the first
// matching entry's Refusal is returned. Blank signatures are skipped (so a
// malformed TOML entry cannot turn every input into a refusal — note that
// strings.Contains(x, "") is true).
func (t *Table) Check(seq string) (Refusal, bool) {
	if t == nil {
		return Refusal{}, false
	}
	norm := normalise(seq)
	if norm == "" {
		return Refusal{}, false
	}
	for i, e := range t.entries {
		for _, sig := range t.normalised[i] {
			if strings.Contains(norm, sig) {
				return Refusal{ID: e.ID, Reason: e.Reason}, true
			}
		}
	}
	return Refusal{}, false
}

// normalise upper-cases s and removes every whitespace rune. It is applied to
// both the input sequence and each signature so FASTA-like wrapping, mixed
// case, and stray tabs cannot mask a match.
func normalise(s string) string {
	return strings.ToUpper(strings.Join(strings.Fields(s), ""))
}
```

```go
package safety

import (
	"encoding/json"
	"strings"
)

// Guard is consulted by the agent loop before executing a tool call. Inspect
// returns a non-zero Refusal and true when the call must be blocked. The
// agent surfaces Refusal.Reason verbatim to the user.
type Guard interface {
	Inspect(toolName string, input json.RawMessage) (Refusal, bool)
}

// NewGuard returns a Guard backed by tbl. A nil tbl produces a guard that
// never refuses (used only in tests; production wiring always supplies a
// loaded table from LoadDefaultTable).
func NewGuard(tbl *Table) Guard { return &guard{tbl: tbl} }

// guard is the concrete Guard.
type guard struct{ tbl *Table }

// Inspect dispatches by tool name. Tools not in the inspected set are passed
// through. Malformed input JSON is treated as "nothing to inspect" — the
// tool itself will surface the parse error in its own error path.
func (g *guard) Inspect(toolName string, input json.RawMessage) (Refusal, bool) {
	if g == nil || g.tbl == nil {
		return Refusal{}, false
	}
	if !inspected(toolName) {
		return Refusal{}, false
	}
	for _, seq := range extractSequences(toolName, input) {
		if r, refused := g.tbl.Check(seq); refused {
			return r, true
		}
	}
	return Refusal{}, false
}

// inspected reports whether toolName falls into a guarded family. The guard
// inspects every design.* tool and lab.submit_experiment. Other lab.* tools
// (cost_estimate, results, etc.) are not inspected: they cannot start a
// real-world experiment by themselves.
func inspected(toolName string) bool {
	if toolName == "lab.submit_experiment" {
		return true
	}
	return strings.HasPrefix(toolName, "design.")
}

// extractSequences pulls every plausibly target-bearing string out of the
// tool's input JSON. It checks three shapes and returns each non-empty
// sequence it finds. Order does not matter; Check short-circuits on the
// first match.
//
//   - target_sequence (string)             — flat top-level key
//   - target.sequence (object -> string)   — nested form
//   - sequences[].sequence (array)         — used by lab.submit_experiment
//     and any design.* tool that takes pre-existing sequences (e.g. as a
//     starting pool for binder design).
func extractSequences(toolName string, input json.RawMessage) []string {
	var doc map[string]json.RawMessage
	if err := json.Unmarshal(input, &doc); err != nil {
		return nil
	}
	var out []string
	if raw, ok := doc["target_sequence"]; ok {
		var s string
		if json.Unmarshal(raw, &s) == nil && s != "" {
			out = append(out, s)
		}
	}
	if raw, ok := doc["target"]; ok {
		var nested struct {
			Sequence string `json:"sequence"`
		}
		if json.Unmarshal(raw, &nested) == nil && nested.Sequence != "" {
			out = append(out, nested.Sequence)
		}
	}
	if raw, ok := doc["sequences"]; ok {
		var seqs []struct {
			Sequence string `json:"sequence"`
		}
		if json.Unmarshal(raw, &seqs) == nil {
			for _, s := range seqs {
				if s.Sequence != "" {
					out = append(out, s.Sequence)
				}
			}
		}
	}
	_ = toolName // reserved for future per-tool-shape branching
	return out
}
```

```go
package agent

import (
	"context"
	"encoding/json"
	"errors"
	"fmt"

	tea "github.com/charmbracelet/bubbletea"

	"github.com/alvarogonjim/proteus/internal/llm"
	"github.com/alvarogonjim/proteus/internal/safety"
	"github.com/alvarogonjim/proteus/internal/tools"
)

// --- bus message types (delivered to the TUI as tea.Msg) ---

// TextDeltaMsg is a chunk of assistant text.
type TextDeltaMsg struct{ Delta string }

// ToolStartMsg announces a tool call is about to run.
type ToolStartMsg struct {
	Name  string
	Input json.RawMessage
}

// ToolDoneMsg reports a finished tool call.
type ToolDoneMsg struct {
	Name    string
	Display string
	Err     error
}

// TurnDoneMsg signals the agent finished its turn. Usage is the turn's total
// token consumption, summed across every LLM call the turn made.
type TurnDoneMsg struct{ Usage llm.Usage }

// TurnErrorMsg reports a fatal error that ended the turn.
type TurnErrorMsg struct{ Err error }

// ConfirmRequestMsg asks the TUI to confirm a sensitive tool call.
type ConfirmRequestMsg struct{ Prompt string }

// ConfirmContextMsg precedes a ConfirmRequestMsg on the bus, carrying the tool
// name and raw input so the TUI can render a tool-specific confirmation (such
// as the rich Adaptyv submit modal) instead of the generic prompt.
type ConfirmContextMsg struct {
	Tool  string
	Input json.RawMessage
}

// ErrBiosecurity is the sentinel error attached to ToolDoneMsg.Err when the
// biosecurity guard refuses a tool call. Callers can match it via
// errors.Is(err, agent.ErrBiosecurity) to distinguish a refusal from a
// generic tool error.
var ErrBiosecurity = errors.New("refused by biosecurity guard")

// Loop is the ReAct agent loop.
type Loop struct {
	provider llm.Provider
	model    string
	registry *tools.Registry
	session  *Session
	bus      chan<- tea.Msg
	confirm  func(prompt string) bool
	guard    safety.Guard
}

// NewLoop builds an agent loop without a biosecurity guard. Production
// callers should use NewLoopWithGuard; this constructor remains for tests and
// for the few code paths (e.g. replay mode in SP-F) where no tools execute.
func NewLoop(p llm.Provider, model string, r *tools.Registry, s *Session,
	bus chan<- tea.Msg, confirm func(string) bool) *Loop {
	return NewLoopWithGuard(p, model, r, s, bus, confirm, nil)
}

// NewLoopWithGuard builds an agent loop that consults guard before every
// tool call. A nil guard disables the check (fail-open in tests only).
func NewLoopWithGuard(p llm.Provider, model string, r *tools.Registry, s *Session,
	bus chan<- tea.Msg, confirm func(string) bool, guard safety.Guard) *Loop {
	return &Loop{
		provider: p, model: model, registry: r, session: s,
		bus: bus, confirm: confirm, guard: guard,
	}
}

// Run executes one user turn: it streams model output, dispatches tool calls,
// and loops until the model stops requesting tools.
func (l *Loop) Run(ctx context.Context, userInput string) {
	l.session.AddUserMessage(userInput)

	var turnUsage llm.Usage

	for {
		if err := ctx.Err(); err != nil {
			l.bus <- TurnErrorMsg{Err: err}
			return
		}

		req := llm.ChatRequest{
			Model:    l.model,
			System:   l.session.SystemPrompt(),
			Messages: l.session.Messages(),
			Tools:    l.registry.Specs(),
		}
		events, err := l.provider.StreamChat(ctx, req)
		if err != nil {
			l.bus <- TurnErrorMsg{Err: err}
			return
		}

		var resp llm.ChatResponse
		for ev := range events {
			switch ev.Kind {
			case "text_delta":
				resp.Text += ev.Delta
				l.bus <- TextDeltaMsg{Delta: ev.Delta}
			case "tool_call":
				resp.ToolCalls = append(resp.ToolCalls, *ev.Call)
			case "done":
				resp.Usage = ev.Usage
				resp.StopReason = ev.StopReason
				turnUsage.InputTokens += ev.Usage.InputTokens
				turnUsage.OutputTokens += ev.Usage.OutputTokens
			case "error":
				l.bus <- TurnErrorMsg{Err: ev.Err}
				return
			}
		}

		l.session.AddAssistantMessage(resp)

		if len(resp.ToolCalls) == 0 {
			l.bus <- TurnDoneMsg{Usage: turnUsage}
			return
		}

		for _, tc := range resp.ToolCalls {
			display := l.executeTool(ctx, tc)
			l.session.AddToolResult(tc.ID, display)
		}
	}
}

// executeTool dispatches one tool call and returns the result text the model
// will see. Errors are returned as text so the model can recover.
func (l *Loop) executeTool(ctx context.Context, tc llm.ToolCall) string {
	input, _ := json.Marshal(tc.Input)
	l.bus <- ToolStartMsg{Name: tc.Name, Input: input}

	tool, ok := l.registry.Get(tc.Name)
	if !ok {
		msg := "error: unknown tool " + tc.Name
		l.bus <- ToolDoneMsg{Name: tc.Name, Display: msg, Err: fmt.Errorf("unknown tool %q", tc.Name)}
		return msg
	}
	if err := ctx.Err(); err != nil {
		l.bus <- ToolDoneMsg{Name: tc.Name, Display: "cancelled", Err: err}
		return "error: cancelled by user"
	}
	if tool.RequiresConfirmation(input) {
		l.bus <- ConfirmContextMsg{Tool: tc.Name, Input: input}
		if !l.confirm("Run " + tc.Name + "? " + string(input)) {
			l.bus <- ToolDoneMsg{Name: tc.Name, Display: "declined by user"}
			return "error: user declined to run " + tc.Name
		}
	}

	// Biosecurity check: consulted AFTER confirmation (the user opted in to
	// running the tool) but BEFORE execution. A refusal here is final — the
	// agent sees the same string we surface to the user, so it stops
	// re-trying with a tweaked target.
	if l.guard != nil {
		if refusal, refused := l.guard.Inspect(tc.Name, input); refused {
			msg := "refused by biosecurity guard: " + refusal.Reason
			l.bus <- ToolDoneMsg{Name: tc.Name, Display: msg, Err: ErrBiosecurity}
			return msg
		}
	}

	res, err := l.registry.Execute(ctx, tc.Name, input)
	if err != nil {
		msg := "error: " + err.Error()
		l.bus <- ToolDoneMsg{Name: tc.Name, Display: msg, Err: err}
		return msg
	}
	l.bus <- ToolDoneMsg{Name: tc.Name, Display: res.Display}
	return res.Display
}
```

## Performance note (v0.5 scaling envelope)

The inspection runs on **every** guarded tool call. With the v0.5 list at ~50 signatures × ~25 residues and input sequences up to ~2000 residues, the cost is N×M `strings.Contains` work — sub-millisecond on the dev box, dwarfed by everything else in `executeTool`. v1+ improvements (k-mer hashing, an Aho-Corasick automaton, or a precompiled regex) are deferred until either the signature list crosses ~10⁴ entries or guard latency shows up in a flame graph.

## Out of scope

Per the spec: distributed updates to the list, per-project overrides, encrypted match tables, k-mer or Aho-Corasick screens, multi-language refusal text, telemetry of refusal events, and any form of "soft" warning short of full refusal. The guard is binary: match → refuse; no match → run.
