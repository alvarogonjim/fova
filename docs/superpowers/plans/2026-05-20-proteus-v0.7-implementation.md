# Proteus v0.7 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. Each numbered Task corresponds to one parallel agent dispatch; the agent receives its task block verbatim as its prompt.

**Spec:** `docs/superpowers/specs/2026-05-20-v0.7-agent-ux-and-grounding.md`

**Goal:** Ship Proteus v0.7 — agent-UX fixes, search/planning hardening, and a container-based local backend that finally runs every tool on the GB10 GPU.

**Architecture:** Four workstreams in two parallel phases. Phase 1 (8 agents) covers TUI polish (slash menu, render), corpus surface fixes, plan-citation validation, search hardening, telemetry plumbing, and the container backend rewrite. Phase 2 (11 agents) layers the system-prompt grounding, stricter plan validation, and a Containerfile-per-tool for all 9 tools. Phase 0 (foundations: spec doc + http wrapper + runtime detect + models cache) is already merged on `master`.

**Tech Stack:** Go 1.22+, Bubble Tea (TUI), uv/Containerfile (installers), NGC PyTorch (`nvcr.io/nvidia/pytorch:25.04-py3` — verify at merge time), Podman or Docker (runtime, auto-detected via `internal/backends/local/runtime.go`).

**Branch convention:** Each task lives on `v0.7/<agent-id>` branched from `origin/master`. Phase 2 agents wait until their Phase 1 dependency merges to master before branching.

**Universal verification (every task):**
1. `go build ./...` — exit 0
2. `go test ./<package(s)-you-touched>/... -race` — all pass
3. `go vet ./...` — exit 0
4. New / modified code has tests (TDD: write the failing test first, then implement)
5. Commit on the named branch with a Conventional Commits message
6. **Do NOT merge to master.** Push the branch and stop. Maintainer reviews and merges in dependency order.

**Universal anti-patterns (every task):**
- Don't add backwards-compatibility shims for code paths nothing calls
- Don't write multi-paragraph docstrings; one-line `// What it does` is the norm in this repo
- Don't log "fixed for v0.7" or "added during the v0.7 refactor" in comments — facts belong in the commit message, not the code

```go
package transport

import (
    "encoding/json"
    "os"
    "path/filepath"
    "sync"
)

// DefaultSink returns an event handler that appends one JSON object per line
// to ~/.proteus/logs/telemetry.jsonl. Errors are swallowed (telemetry must
// never crash the caller).
func DefaultSink() func(Event) {
    home, _ := os.UserHomeDir()
    return newSink(filepath.Join(home, ".proteus", "logs", "telemetry.jsonl"))
}

func newSink(path string) func(Event) {
    var mu sync.Mutex
    return func(e Event) {
        mu.Lock()
        defer mu.Unlock()
        _ = os.MkdirAll(filepath.Dir(path), 0o755)
        f, err := os.OpenFile(path, os.O_APPEND|os.O_CREATE|os.O_WRONLY, 0o644)
        if err != nil { return }
        defer f.Close()
        raw, err := json.Marshal(e)
        if err != nil { return }
        f.Write(raw)
        f.Write([]byte{'\n'})
    }
}
```
