**Goal:** Replace the thin `fold.boltz2` tool with a real Boltz-2 integration — typed multi-entity schema, MSA control, model parameters, input preflight, and confidence-score ingestion.

**Tech Stack:** Go 1.22+, the `tools.Tool` interface, `jobs.Manager`, the local container backend (podman/docker), Boltz-2 (`boltz predict`) in a tool image.

## THE CONTRACT — pinned, do not deviate

Streams A and B both parse the same tool-call JSON. Both stream agents copy these definitions **verbatim** into their package. Drift here is the only way the streams can break each other.

### JSON request shape

```json
{
  "entities": [
    {"type": "protein", "id": "A", "sequence": "MKQ...", "msa": "empty", "cyclic": false},
    {"type": "protein", "id": ["B", "C"], "sequence": "AAA..."},
    {"type": "ligand", "id": "L", "smiles": "CCO"}
  ],
  "recycling_steps": 3,
  "sampling_steps": 200,
  "diffusion_samples": 1,
  "step_scale": 1.638,
  "output_format": "pdb",
  "save_as": "designs/predicted.pdb"
}
```

### Shared Go types (identical in both packages)

```go
// chainIDs unmarshals a JSON chain id given either as a string ("A") or a
// string array (["B","C"]) into a uniform []string.
type chainIDs []string

func (c *chainIDs) UnmarshalJSON(b []byte) error {
	var one string
	if err := json.Unmarshal(b, &one); err == nil {
		*c = chainIDs{one}
		return nil
	}
	var many []string
	if err := json.Unmarshal(b, &many); err != nil {
		return fmt.Errorf("id must be a string or a list of strings: %w", err)
	}
	*c = chainIDs(many)
	return nil
}

// boltz2Entity is one molecular component of the predicted complex.
type boltz2Entity struct {
	Type     string   `json:"type"`     // protein | dna | rna | ligand
	ID       chainIDs `json:"id"`       // one or more chain ids
	Sequence string   `json:"sequence"` // protein/dna/rna
	SMILES   string   `json:"smiles"`   // ligand (exclusive with ccd)
	CCD      string   `json:"ccd"`      // ligand (exclusive with smiles)
	MSA      string   `json:"msa"`      // "empty" | "server" | workspace path; protein/dna/rna
	Cyclic   bool     `json:"cyclic"`   // protein/dna/rna
}

// boltz2Request is the full fold.boltz2 input. Pointer fields are model
// parameters: nil ⇒ omit the CLI flag and let Boltz-2 use its own default.
type boltz2Request struct {
	Entities         []boltz2Entity `json:"entities"`
	RecyclingSteps   *int           `json:"recycling_steps"`
	SamplingSteps    *int           `json:"sampling_steps"`
	DiffusionSamples *int           `json:"diffusion_samples"`
	StepScale        *float64       `json:"step_scale"`
	OutputFormat     string         `json:"output_format"` // "pdb" (default) | "mmcif"
	SaveAs           string         `json:"save_as"`
}
```

Stream A names its copy `boltz2Request` in package `fold`. Stream B keeps the name `boltz2Request` in package `local` (replacing the existing one). Same JSON tags ⇒ same wire format.

### Entity type rules (both streams enforce; A in preflight, B defensively)

- `type` ∈ {`protein`, `dna`, `rna`, `ligand`}.
- `protein`/`dna`/`rna`: non-empty `sequence`. `ligand`: exactly one of `smiles`/`ccd`.
- `msa` and `cyclic` apply to `protein`/`dna`/`rna` only.
- `msa` ∈ {`""`, `empty`, `server`, a workspace path}. Empty string is treated as `empty`.
- chain ids are unique across all entities.

### YAML the adapter emits (Stream B, exact format)

```
version: 1
sequences:
  - protein:
      id: A
      sequence: MKQ...
      msa: empty
  - protein:
      id: [B, C]
      sequence: AAA...
  - ligand:
      id: L
      smiles: CCO
```

Rules: `version: 1` first. One list item per entity, in input order. Item key = entity `type`. `id` scalar when one id, flow list `[B, C]` when several. `sequence` for protein/dna/rna; `smiles` or `ccd` for ligand. `msa: empty` line when `msa` is `""`/`empty`; `msa: <staged-path>` when a file; **no `msa` line** when `server`. `cyclic: true` line only when `Cyclic` is true. Two-space indent per level.

### Score keys the adapter produces (Stream B)

From each `confidence_in_model_<N>.json`, populate `designOut.Scores`:

| Score key | Source field |
|---|---|
| `plddt` | `complex_plddt` |
| `ptm` | `ptm` |
| `iptm` | `iptm` |
| `confidence_score` | `confidence_score` |
| `ligand_iptm` | `ligand_iptm` |
| `protein_iptm` | `protein_iptm` |
| `complex_iplddt` | `complex_iplddt` |
| `complex_pde` | `complex_pde` |
| `complex_ipde` | `complex_ipde` |
| `chain_<k>_ptm` | `chains_ptm[<k>]` for each key k |

A missing/unparseable confidence file ⇒ empty `Scores`, **no error**.
