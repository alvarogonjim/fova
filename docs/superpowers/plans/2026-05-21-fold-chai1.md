> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the thin `fold.chai1` tool with a real Chai-1 integration — typed multi-entity schema, restraints, templates, MSA control, model parameters, input preflight, and `.npz` confidence-score ingestion — and retire the now-unused `foldJobTool`.

**Architecture:** Three file-disjoint streams built in parallel worktrees, then merged. Stream A is the bespoke agent tool (`internal/tools/fold`) and retires `foldJobTool`; Stream B is the local-backend adapter plus a small `.npz` reader (`internal/backends/local`); Stream C is the grounding skill. Streams A and B never share a file — they communicate only through the **JSON request contract** pinned in this plan.

**Tech Stack:** Go 1.22+, the `tools.Tool` interface, `jobs.Manager`, the local container backend, Chai-1 (`chai-lab fold`) in a tool image.

## THE CONTRACT — pinned, do not deviate

Streams A and B both parse the same tool-call JSON. Both copy these definitions **verbatim** into their package.

### JSON request shape

```json
{
  "entities": [
    {"type": "protein", "id": "A", "sequence": "MKQ..."},
    {"type": "ligand",  "id": "L", "smiles": "CCO"},
    {"type": "rna",     "id": "R", "sequence": "ACGU"},
    {"type": "glycan",  "id": "G", "glycan": "NAG(4-1 NAG)"}
  ],
  "msa": "default",
  "restraints": [
    {"connection_type": "contact", "chain_a": "A", "res_a": "A219",
     "chain_b": "L", "res_b": "", "min_distance": 0.0, "max_distance": 6.0,
     "confidence": 1.0, "comment": ""}
  ],
  "templates": {"server": false, "hits_path": ""},
  "num_trunk_recycles": 3,
  "num_diffn_timesteps": 200,
  "num_diffn_samples": 5,
  "num_trunk_samples": 1,
  "recycle_msa_subsample": 0,
  "seed": 42,
  "save_as": "out/predicted.cif"
}
```

### Shared Go types (identical in both packages)

```go
// chai1Entity is one molecular component of the predicted complex.
type chai1Entity struct {
	Type     string `json:"type"`     // protein | dna | rna | ligand | glycan
	ID       string `json:"id"`       // one chain id
	Sequence string `json:"sequence"` // protein/dna/rna
	SMILES   string `json:"smiles"`   // ligand
	Glycan   string `json:"glycan"`   // glycan
}

// chai1Restraint is one inter-chain restraint. Pointer fields are optional
// numerics: nil ⇒ the CSV cell is left blank.
type chai1Restraint struct {
	ConnectionType string   `json:"connection_type"` // contact | pocket | covalent
	ChainA         string   `json:"chain_a"`
	ResA           string   `json:"res_a"`
	ChainB         string   `json:"chain_b"`
	ResB           string   `json:"res_b"`
	MinDistance    *float64 `json:"min_distance"`
	MaxDistance    *float64 `json:"max_distance"`
	Confidence     *float64 `json:"confidence"`
	Comment        string   `json:"comment"`
}

// chai1Templates is the optional request-level template configuration.
type chai1Templates struct {
	Server   bool   `json:"server"`
	HitsPath string `json:"hits_path"`
}

// chai1Request is the full fold.chai1 input. Pointer fields are model
// parameters: nil ⇒ omit the CLI flag and let Chai-1 use its own default.
type chai1Request struct {
	Entities            []chai1Entity    `json:"entities"`
	MSA                 string           `json:"msa"` // "default" | "server" | workspace path
	Restraints          []chai1Restraint `json:"restraints"`
	Templates           *chai1Templates  `json:"templates"`
	NumTrunkRecycles    *int             `json:"num_trunk_recycles"`
	NumDiffnTimesteps   *int             `json:"num_diffn_timesteps"`
	NumDiffnSamples     *int             `json:"num_diffn_samples"`
	NumTrunkSamples     *int             `json:"num_trunk_samples"`
	RecycleMSASubsample *int             `json:"recycle_msa_subsample"`
	Seed                *int             `json:"seed"`
	SaveAs              string           `json:"save_as"`
}
```

### Entity rules (A enforces in preflight, B defensively)

- `type` ∈ {`protein`, `dna`, `rna`, `ligand`, `glycan`}.
- `protein`/`dna`/`rna`: non-empty `Sequence`. `ligand`: non-empty `SMILES`. `glycan`: non-empty `Glycan`.
- chain `id`s unique across all entities.
- `msa` ∈ {`""`, `default`, `server`, a workspace path}; `""` is treated as `default`.

### FASTA the adapter emits (Stream B, exact format)

```
>protein|name=A
MKQ...
>ligand|name=L
CCO
>rna|name=R
ACGU
>glycan|name=G
NAG(4-1 NAG)
```

One record per entity in input order: `">" + type + "|name=" + id + "\n"` then the body line (`Sequence` for protein/dna/rna, `SMILES` for ligand, `Glycan` for glycan) + `"\n"`.

### Restraint CSV the adapter emits (Stream B, exact format)

A header row then one row per restraint. Columns, in order:

```
restraint_id,chainA,res_idxA,chainB,res_idxB,min_distance_angstrom,max_distance_angstrom,connection_type,confidence,comment
```

- `restraint_id` — generated: `restraint_0`, `restraint_1`, … (row index).
- `chainA`←`ChainA`, `res_idxA`←`ResA`, `chainB`←`ChainB`, `res_idxB`←`ResB`, `connection_type`←`ConnectionType`, `comment`←`Comment` — written verbatim.
- `min_distance_angstrom`/`max_distance_angstrom` — the float when the pointer is set, blank cell when nil.
- `confidence` — the float when set, `1.0` when nil.

The file is written only when `len(Restraints) > 0`.

### Score keys the adapter produces (Stream B)

From each `scores.model_idx_N.npz`, populate `designOut.Scores`:

| Score key | npz key | shape |
|---|---|---|
| `aggregate_score` | `aggregate_score` | scalar |
| `ptm` | `ptm` | scalar |
| `iptm` | `iptm` | scalar |
| `chain_<i>_ptm` | `per_chain_ptm[i]` | 1-D, flattened |
| `has_inter_chain_clashes` | `has_inter_chain_clashes` | scalar (0/1) |

2-D arrays (`per_chain_pair_iptm`, `chain_chain_clashes`) are skipped. A missing/unreadable `.npz` ⇒ empty `Scores`, **no error**.

### The npz reader API (Stream B internal, pinned)

```go
// npzValue is one array decoded from a .npz member. Data is the flattened
// values (scalars, 1-D, or higher). Shape is the numpy shape — len 0 is a
// scalar, len 1 a vector. All dtypes are widened to float64.
type npzValue struct {
	Shape []int
	Data  []float64
}

// readNPZ opens a .npz (a ZIP of .npy members) and decodes each member.
func readNPZ(path string) (map[string]npzValue, error)
```

### Task B1: The `.npz` reader

**Files:**
- Create: `internal/backends/local/npz.go`
- Test: `internal/backends/local/npz_test.go` (create)

A `.npz` file is a ZIP archive whose members are `.npy` files. A `.npy` file is: 6-byte magic `\x93NUMPY`; 1 byte major version, 1 byte minor; a header-length field (major 1 → 2-byte little-endian `uint16`; major ≥ 2 → 4-byte little-endian `uint32`); then an ASCII Python-dict header (`{'descr': '<f8', 'fortran_order': False, 'shape': (3,), }`, space-padded, `\n`-terminated); then the raw little-endian array data. `descr` gives the dtype, `shape` the dimensions.

// makeNPY builds a valid version-1.0 .npy byte slice for a float64 array.
// shapeText is the numpy shape tuple: "()" scalar, "(3,)" a 3-vector.
func makeNPY(shapeText string, vals []float64) []byte {
	hdr := "{'descr': '<f8', 'fortran_order': False, 'shape': " + shapeText + ", }"
	total := 10 + len(hdr) + 1 // 6 magic + 2 version + 2 len + header + \n
	if pad := (64 - total%64) % 64; pad > 0 {
		hdr += strings.Repeat(" ", pad)
	}
	hdr += "\n"
	var b bytes.Buffer
	b.WriteString("\x93NUMPY")
	b.WriteByte(1)
	b.WriteByte(0)
	_ = binary.Write(&b, binary.LittleEndian, uint16(len(hdr)))
	b.WriteString(hdr)
	for _, v := range vals {
		_ = binary.Write(&b, binary.LittleEndian, v)
	}
	return b.Bytes()
}

func makeNPZ(t *testing.T, members map[string][]byte) string {
	t.Helper()
	path := filepath.Join(t.TempDir(), "scores.npz")
	f, err := os.Create(path)
	if err != nil {
		t.Fatal(err)
	}
	defer f.Close()
	zw := zip.NewWriter(f)
	for name, data := range members {
		w, err := zw.Create(name + ".npy")
		if err != nil {
			t.Fatal(err)
		}
		if _, err := w.Write(data); err != nil {
			t.Fatal(err)
		}
	}
	if err := zw.Close(); err != nil {
		t.Fatal(err)
	}
	return path
}
